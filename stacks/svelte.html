Svelte + SvelteKit

Full .svelte SFC support. Script extraction for logic analysis, template scanning for security + accessibility.

Quick Start

npx @vibecodeqa/cli

Set up CI + auto-fix

npx @vibecodeqa/cli init    # creates .github/workflows/vibecodeqa.yml
npx @vibecodeqa/cli fix     # auto-fix lint issues + show fix suggestions

Auto-detects: svelte or @sveltejs/kit in dependencies, Vite bundler, vitest test runner.

How .svelte Files Are Analyzed

Part of SFCWhat's Checked
<script>Type safety, complexity, duplication, error handling, imports, architecture graph
<script context="module">Same — both script blocks are extracted and concatenated
Template (HTML)Accessibility (img alt, click handlers), Security ({@html} XSS)
<style>Not analyzed (CSS is out of scope)

Svelte-Specific Patterns

Security

PatternSeverityWhy
{@html ...}Warning (CWE-79)Renders raw HTML — equivalent to React's dangerouslySetInnerHTML. Sanitize with DOMPurify.

Accessibility

PatternSeverityWhy
<div on:click> without role + on:keydownWarningNon-interactive elements with click handlers exclude keyboard users
<img> without altErrorImages without alt text are invisible to screen readers

SvelteKit Detection

SvelteKit is detected via @sveltejs/kit in dependencies:

  • Framework: "svelte", Bundler: "vite"
  • src/routes/ included in source scanning (under src/)
  • .svelte-kit/ excluded from scanning
