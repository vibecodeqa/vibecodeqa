# Standards Examples

Examples show how standards coverage applies to real repositories. They are intentionally
more concrete than the stack charters: each example names repo slices, current authored
coverage, unresolved gaps, and the next standard to author.

## Examples

- [Cloudflare SaaS app coverage](cloudflare-saas-app.md): maps a real
  React/Cloudflare/D1/MCP repository to React SPA, Cloudflare Pages Fullstack, D1, MCP
  Worker, CLI, and SDK standards.

## How to read these pages

- **Covered** means there is an authored versioned rubric that a repo can be judged
  against today.
- **Planned** means there is a stack charter or registry entry, but the full rubric still
  needs to be authored.
- **Gap** means the resolver found a stack item or composed standard the repo needs, but
  VCQA does not yet have enough standard text to judge it consistently.
