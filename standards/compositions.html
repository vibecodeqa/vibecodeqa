Standards composition map

VCQA standards are built from upstream authorities plus the stack-specific glue needed to judge real repos. This page starts with individual stack items, then composes them into standards we should author.

Decision. We do not author a generic "React standard" or "TypeScript standard". React, TypeScript, WCAG, OWASP, Cloudflare, and MCP already publish the broad doctrine. VCQA standards define deployable stack shapes and cite those sources.

Machine-readable version: /standards/compositions.json.

Authoring Order

Start with the stack standards that have the highest reuse and the clearest repo evidence:

  • cloudflare-pages-fullstack: unlocks CRM's app/functions shape.
  • cloudflare-d1-app: captures migration and tenant database discipline.
  • cloudflare-worker-mcp-server: captures MCP over Workers, Durable Objects, OAuth, and tool safety.
  • node-cli-internal-tool, typescript-sdk, and zensical-kb-site: cover the shared supporting packages and documentation model.

This is the composition plan. Authored rubrics live under /standards/; source authorities live in References.
