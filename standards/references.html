External standards registry

Primary sources VibeCode QA should cite when authoring stack-specific gold-standard KBs. This page is the authority map: official specifications, framework docs, platform docs, security standards, and testing guides.

Authoring rule. Do not re-create broad framework doctrine when an upstream standard already exists. VCQA standards compose these sources, then add only the stack shape, deployment constraints, detection mapping, exceptions, and anti-patterns needed to judge a repository.

Machine-readable version: /standards/references.json.
